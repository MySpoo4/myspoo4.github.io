2. System‚Äôs Description & Core Features üíª

Built with Django and enhanced with modern frontend technologies, StudyBuddy offers a seamless and engaging user experience. Sprint 1 focused on establishing the foundational features for user interaction and matching.

Core Sprint 1 Features

Compatibility-Based Matching: Users are presented with potential matches sorted by a compatibility score based with potential matches sorted by a compatibility score based on shared courses and study preferences.

Interactive Swipe Interface: A dynamic, card-based UI allows users to "like" or "pass" on profiles, with smooth animations.

Real-Time Match Notifications: When two users mutually like each other, a "match" is created, and an instant on-screen popup notifies the user who completed the match.

Technical Highlights

Backend: Django, handling user authentication, database models (Profile, Swipe, Match), and business logic.

Frontend: Django Templates with TailwindCSS for responsive design.

Asynchronous Actions: JavaScript's Fetch API is used for swipe actions and polling for new match notifications without page reloads.

3. Design Rationale & Architecture ‚öôÔ∏è

The architecture follows Django's Model-View-Template (MVT) pattern. A key design choice was to offload the swipe and notification logic to the client-side using JavaScript, communicating with dedicated Django API endpoints. This creates a responsive, app-like feel. The compatibility scoring is handled efficiently on the backend to ensure the most relevant profiles are shown first.
