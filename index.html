2. System‚Äôs Description & Core Features üíª (Sprint 2)

Sprint 2 extended the Django backend and dynamic frontend to support post-match interactions and administrative oversight. The focus shifted from discovery to communication, safety, and quality control, making the platform more mature and user-centric.

Core Sprint 2 Features

In-App Chat: Matched users can initiate private conversations to coordinate study sessions without leaving the platform.

Session Ratings & Feedback: After a session, users can rate their experience and provide feedback, contributing to a more reliable matching ecosystem.

Enhanced Privacy Controls: Students have granular control over what profile information and location data is visible to others.

Admin & Moderation

Analytics Dashboard: Admins can view key metrics like user signups, total matches, and active sessions to monitor platform health and engagement.

Moderation Tools: A dedicated interface for admins to manage user reports, review content, and suspend accounts to ensure a safe environment.

3. Design Rationale & Architecture ‚öôÔ∏è (Sprint 2)

The architecture was extended to support real-time communication and administrative functions. The chat system was built using asynchronous JavaScript calls to Django REST endpoints, enabling instant messaging. The admin dashboard and moderation tools were integrated into Django's admin panel for security and rapid development. New models for Chat, Rating, and Report were added to the database, with business logic handled in their respective views to maintain a clean MVT structure.
